## Описание работы

В этой лабораторной работе я продолжил работать с предметной областью спортсменов.

ЛР-5 строится на основе прошлых лабораторных:

## Стратегии сортировки

В файле `strategies.py` я реализовал функции-стратегии сортировки:

1) `by_name()` - сортировка по имени;
2) `by_age()` - сортировка по возрасту;
3) `by_training_load()` - сортировка по тренировочной нагрузке;
4) `by_name_age()` - сортировка сначала по возрасту, потом по имени.

Эти функции передаются в метод `sort_by()`:

`collection.sort_by(by_name)`

Главная идея: коллекция сама не знает, как именно сортировать объекты. Мы передаём ей функцию, которая говорит, по какому признаку сортировать.

## Функции-фильтры

Также были реализованы функции-фильтры:

1) `is_active()` - оставляет только активных спортсменов;
2) `is_runner()` - оставляет только бегунов;
3) `is_swimmer()` - оставляет только пловцов.

Фильтрация была показана двумя способами:

1) через встроенную функцию `filter()`;
2) через метод коллекции `filter_by()`.

Например:

`collection.filter_by(is_runner)`

## Использование map()

С помощью `map()` я преобразовал коллекцию объектов в список имён спортсменов.

Например:

`names = list(map(lambda athlete: athlete.name, collection))`

Также через `map()` объекты были преобразованы в строки.

## Lambda

В работе использовалась `lambda` для простой одноразовой функции.

Например:

`collection.sort_by(lambda athlete: athlete.name)`

Также я сравнил этот вариант с обычной именованной функцией:

`collection.sort_by(by_name)`

Оба варианта дают одинаковый результат.

## Фабрики функций

В лабораторной были реализованы фабрики функций.

Фабрика функций - это функция, которая создаёт и возвращает другую функцию.

Я сделал:

1) `make_min_age_filter()` - создаёт фильтр по минимальному возрасту;
2) `make_min_training_load_filter()` - создаёт фильтр по минимальной тренировочной нагрузке.

Например:

`min_age_filter = make_min_age_filter(21)`

После этого фильтр можно передать в коллекцию:

`collection.filter_by(min_age_filter)`

## Callable-объекты и паттерн «Стратегия»

Для задания на 5 я реализовал callable-объекты:

1) `AddVisitsStrategy` - добавляет спортсмену заданное количество посещений;
2) `SetPaidStrategy` - меняет статус оплаты на `True`.

Callable-объект - это объект, который можно вызвать как функцию. Для этого используется метод `__call__()`.

Например:

`add_visits = AddVisitsStrategy(2)`

`collection.apply(add_visits)`

Это показывает паттерн «Стратегия»: можно менять поведение программы, просто передавая другую функцию или объект-стратегию.

## Коллекция Lab05AthleteCollection

В файле `collection.py` была создана коллекция `Lab05AthleteCollection`.

Она наследуется от коллекции из ЛР-3 и добавляет методы:

1) `sort_by(key_func)` - сортировка по переданной функции;
2) `filter_by(predicate)` - фильтрация по переданной функции;
3) `apply(func)` - применение функции ко всем элементам коллекции;
4) `show()` - вывод коллекции.

Главное отличие этой коллекции в том, что она не содержит жёстко прописанных способов сортировки и фильтрации. Все правила передаются снаружи.

## Цепочка операций

Также была реализована цепочка операций:

`collection.filter_by(is_active).sort_by(by_age_and_name).apply(deactivate_athlete)`

В этой цепочке:

1) сначала остаются только активные спортсмены;
2) потом они сортируются по возрасту и имени;
3) потом ко всем применяется функция деактивации.
