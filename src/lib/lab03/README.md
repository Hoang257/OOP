## Описание иерархии классов:
Создал два дочерних класса (Runner, Swimmer), которые наследуют от родительского (Athelete)

В базовом классе те же методы, что и в лабораторной работе №1, а в дочерних я добавил метод training_load() - общий метод для расчёта тренировочной нагрузки, чтобы реализовать полиморфизм и display() - вывод информации об объекте.

В классе Runner были добавлены доп. атрибуты:

1) distance - дистанция бега.
2) best_time - лучшее время на дистанции.

и доп. метод:

1) average_speed() - рассчитывает среднюю скорость бегуна.

Переопределённые методы:

1) can_compete_with() - бегун может соревноваться только с другим бегуном на такой же дистанции.
2) training_load() — тренировочная нагрузка бегуна рассчитывается с учётом дистанции.
3) __str__() — выводит информацию о бегуне с дополнительными полями.

В классе Swimmer были добавлены доп. атрибуты:

1) pool_distance - дистанция бассейна.
2) best_time - лучшее время.
3) swimming_style - стиль плавания.

и доп. метод:

1) swim_distance() — рассчитывает общую дистанцию плавания по количеству кругов.

Переопределённые методы:

1) can_compete_with() — пловец может соревноваться только с другим пловцом с таким же стилем плавания и такой же дистанцией бассейна;
2) training_load() — тренировочная нагрузка пловца рассчитывается с учётом дистанции бассейна;
3) __str__() — выводит информацию о пловце с дополнительными полями.

Для интеграции с ЛР-2 была создана коллекция Lab03AthleteCollection, которая наследуется от коллекции AthleteCollection из ЛР-2.

Добавиk взаимодействие с коллекцией:

1) фильтрация по типу

2) выборка только одного вида объектов

Какие методы ?
1) get_only_runners() - возвращает коллекцию только с бегунами;
2) get_only_swimmers() - возвращает коллекцию только с пловцами;
3) show_training_loads() - вызывает метод training_load()
