## Аннотации типов

В ЛР-6 используются аннотации типов.

Аннотации нужны для того, чтобы было понятно, какие типы данных принимает функция и что она возвращает.

Например:

```python
def add(self, item: T) -> None:
```

Это означает, что метод `add()` принимает объект типа `T` и ничего не возвращает.

Также были использованы аннотации:

1) `int` - целое число;
2) `str` - строка;
3) `float` - число с точкой;
4) `bool` - логическое значение;
5) `list[T]` - список объектов типа `T`;
6) `Optional[T]` - объект типа `T` или `None`;
7) `Callable` - функция, которую можно передать как аргумент.

## Generic и TypeVar

В файле `container.py` была создана типизированная коллекция:

```python
class TypedCollection(Generic[T]):
```

Здесь `T` - это переменная типа.

Она создаётся так:

```python
T = TypeVar("T")
```

Это значит, что коллекция может работать с разными типами объектов.

Например:

```python
runners: TypedCollection[Runner] = TypedCollection(Runner)
```

Такую коллекцию можно понимать как коллекцию бегунов.

Если в неё добавить объект класса `Runner`, всё будет нормально.

Если попытаться добавить `Swimmer`, будет ошибка, потому что коллекция была создана для объектов `Runner`.

## Методы find, filter и map

Для задания на 4 были добавлены методы:

1) `find(predicate)` - возвращает первый подходящий объект или `None`;
2) `filter(predicate)` - возвращает список всех подходящих объектов;
3) `map(transform)` - применяет функцию к каждому объекту и возвращает список результатов.

Пример:

```python
found_runner = runners.find(lambda runner: runner.name == "Маша")
```

Если объект найден, метод вернёт его.

Если объект не найден, метод вернёт `None`.

Пример `filter()`:

```python
filtered_runners = runners.filter(lambda runner: runner.age >= 22)
```

Этот код вернёт список бегунов от 22 лет.

Пример `map()`:

```python
names = runners.map(lambda runner: runner.name)
```

Была коллекция `Runner`, а получился список строк `list[str]`.

Также был пример:

```python
loads = runners.map(lambda runner: runner.score())
```

Здесь была коллекция `Runner`, а получился список чисел `list[float]`.

Для этого используется второй TypeVar:

```python
R = TypeVar("R")
```

Он нужен потому, что результат `map()` может быть другого типа.

## Protocol

Для задания на 5 были созданы два Protocol:

1) `Displayable`;
2) `Scorable`.

`Protocol` описывает, что объект должен уметь делать.

Класс не обязан явно наследоваться от Protocol. Достаточно, чтобы у него был нужный метод.

## Protocol Displayable

Протокол `Displayable` требует метод:

```python
display()
```

Если у объекта есть метод `display()`, значит он подходит под `Displayable`.

В моей системе `Runner` и `Swimmer` подходят под этот протокол, потому что у них есть метод `display()`.

## Protocol Scorable

Протокол `Scorable` требует метод:

```python
score()
```

Если у объекта есть метод `score()`, значит он подходит под `Scorable`.

Метод `score()` был добавлен в базовый класс ЛР-3 и возвращает тренировочную нагрузку как число.

## runtime_checkable

Для протоколов был использован декоратор:

```python
@runtime_checkable
```

Он нужен, чтобы можно было проверять объект через `isinstance()` во время выполнения программы.

Например:

```python
isinstance(runner1, Displayable)
```

Без `@runtime_checkable` такая проверка с Protocol не работала бы нормально.

## TypeVar с ограничением bound

Также были созданы TypeVar с ограничением:

```python
D = TypeVar("D", bound=Displayable)
S = TypeVar("S", bound=Scorable)
```

`D` означает любой тип, который подходит под `Displayable`.

То есть объект должен иметь метод `display()`.

`S` означает любой тип, который подходит под `Scorable`.

То есть объект должен иметь метод `score()`.

Это нужно для методов:

```python
display_all()
get_scores()
```

В методе `display_all()` можно безопасно вызвать:

```python
item.display()
```

А в методе `get_scores()` можно безопасно вызвать:

```python
item.score()
```
