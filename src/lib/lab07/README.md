# ЛР-7 — консольное приложение

Интерактивное CLI для коллекции спортсменов (модели **ЛР1–3**, коллекция и стратегии **ЛР5**). Код предметной области в `src/lib/lab01` … `lab05` **не изменяется**.

## Запуск

```bash
cd src/lib/lab07
python main.py
```

Или **F5** в VS Code (конфигурация «ЛР-7» в `.vscode/launch.json`), либо двойной щелчок по `run.bat`.

Данные: `data/athletes.json` (создаётся/обновляется при выходе, пункт **0**).

## Структура (слои)

| Файл | Назначение |
|------|------------|
| `main.py` | Точка входа, `sys.path` → корень `OOP` |
| `cli.py` | Меню, ввод/вывод, `try/except` |
| `app.py` | Бизнес-логика, коллекция, поиск, сортировка |
| `models.py` | Реэкспорт `Athlete`, `Runner`, `Swimmer` |
| `storage.py` | `save()` / `load()` — JSON |
| `exceptions.py` | `DuplicateAthleteError`, `AthleteNotFoundError` |

## Меню (по заданию)

1. Добавить (атлет / бегун / пловец)  
2. Показать всех (таблица)  
3. Найти по ID  
4. Найти по имени  
5. Поиск по атрибуту  
6. Фильтрация (активные, возраст, нагрузка, бегуны, пловцы)  
7. Сортировка: имя / нагрузка («цена») / дата добавления  
8. Изменить число посещений  
9. Удалить по ID (`y/n`)  
0. Выход + автосохранение  

## Соответствие ТЗ

- Разделение **CLI → app → models/storage**  
- Собственные исключения, перехват в CLI  
- Ввод пункта меню: `int(input(...))` + `except ValueError`  
- Оценка **5**: JSON, типизация в `app.py`, дата добавления в файле (не в классах)  

## Демонстрация работы

Одна запись **asciinema** должна показать все четыре сценария ниже (оценка на **5** — те же сценарии, что и для скриншотов).

### Сценарии (что показать в записи)

| № | Сценарий | Действия в CLI |
|---|----------|----------------|
| 1 | Старт и загрузка | Запуск `python main.py` → сообщение о файле данных → **2** (таблица; можно пустую) |
| 2 | CRUD и сохранение | **1** добавить бегуна и пловца → **2** → **9** удалить одного (`y`) → **0** → снова `python main.py` → **2** (данные сохранились) |
| 3 | Сортировка | **7** → по имени → **2**; снова **7** → по нагрузке → **2**; **7** → по дате добавления → **2** |
| 4 | Фильтр и ошибки | **6** (например, только бегуны) → **1** с тем же **ID** (дубликат) → **3** с несуществующим ID → в меню ввести `abc` (неверный пункт) |
